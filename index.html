User Guide OpenStack Cloud

Ambiente: CNR Biomics Cloud (OpenStack)

1. Introduzione

Questa guida descrive l’utilizzo della piattaforma OpenStack Cloud per l’ambiente CNR Biomics. OpenStack è una piattaforma cloud open-source che consente la gestione di risorse virtuali come macchine, storage e reti.

2. Accesso alla piattaforma

L’accesso avviene tramite dashboard Horizon o API.

2.1 Accesso via Dashboard (Horizon)

  • Aprire il browser
  • Inserire l’URL del portale OpenStack (fornito dall’amministratore)
  • Effettuare login con username e password istituzionali

2.2 Accesso via CLI

Installare il client OpenStack:

pip install openstackclient

Caricare le credenziali:

source openrc.sh

3. Concetti principali

  • Project (Tenant): ambiente isolato di lavoro
  • Instance: macchina virtuale
  • Flavor: risorse hardware (CPU, RAM, disco)
  • Image: sistema operativo base
  • Network: rete virtuale
  • Security Group: firewall virtuale
  • Volume: storage persistente

4. Creazione di una macchina virtuale (Instance)

  1. Accedere alla dashboard
  2. Selezionare Compute → Instances
  3. Cliccare su Launch Instance
  4. Inserire:
    • Nome istanza
    • Image (es. Ubuntu, CentOS)
    • Flavor (CPU/RAM)
    • Network
  5. Configurare chiavi SSH
  6. Avviare l’istanza
È consigliato utilizzare chiavi SSH invece di password per motivi di sicurezza.

5. Gestione delle reti

Le reti in OpenStack permettono la comunicazione tra istanze e verso l’esterno.

  • Creazione rete privata
  • Subnetting
  • Router verso rete pubblica

6. Security Groups

I Security Groups definiscono le regole firewall per le istanze.

  • Permettere SSH (porta 22)
  • Permettere HTTP (porta 80)
  • Permettere HTTPS (porta 443)
Evitare di aprire tutte le porte verso Internet (0.0.0.0/0) senza necessità.

7. Storage (Volumes)

I volumi sono dischi persistenti separati dalle istanze.

  1. Creare volume da Volumes
  2. Selezionare dimensione
  3. Attaccare alla VM

8. Accesso alle istanze

Connessione tramite SSH:

ssh -i key.pem ubuntu@IP_PUBLICO

Assicurarsi che la porta 22 sia aperta nei Security Groups.

9. Gestione tramite CLI

Lista istanze:

openstack server list

Creazione istanza:

openstack server create --image IMAGE --flavor FLAVOR --network NETWORK NAME

10. Troubleshooting

  • Instance non raggiungibile: controllare Security Group e Floating IP
  • No SSH access: verificare chiave e permessi
  • Errore di deploy: controllare quota progetto

11. Best Practices

  • Usare sempre SSH keys
  • Limitare accessi di rete
  • Monitorare uso risorse
  • Eliminare risorse inutilizzate

12. Supporto

Per supporto tecnico contattare l’amministratore del cloud CNR Biomics o il team IT interno.
